    ## Tailwind CSS !important modifier

    <TIL.ItemTags>
      <TIL.ItemTag>tailwindcss</TIL.ItemTag>
      <TIL.ItemTag>css</TIL.ItemTag>
    </TIL.ItemTags>

    Make any utility important by adding a `!` character to the beginning:

    ```html
    <p class="font-bold !font-medium">
      This will be medium even though bold comes later in the CSS.
    </p>
    ```

    [Official Docs](https://tailwindcss.com/docs/configuration#important-modifier)

  </TIL.Item>

    ## pre overflow: scroll inside flex

    <TIL.ItemTags>
      <TIL.ItemTag>css</TIL.ItemTag>
      <TIL.ItemTag>pre</TIL.ItemTag>
      <TIL.ItemTag>flex</TIL.ItemTag>
    </TIL.ItemTags>

    Ever used `overflow-x: scroll` on a `pre` element but it didn't work, and the parent had the scrollbar instead?

    Maybe the parent is inside a `display: flex`. If so, try adding `min-width: 0` to the parent.

    [Credit](https://weblog.west-wind.com/posts/2016/feb/15/flexbox-containers-pre-tags-and-managing-overflow)

    ## :not() pseudo-class

    <TIL.ItemTags>
      <TIL.ItemTag>css</TIL.ItemTag>
      <TIL.ItemTag>selector</TIL.ItemTag>
    </TIL.ItemTags>

    Using `:not()` is quite tricky, using it incorrectly can yield unexpected results.

    Example, styling inline `code` that is not a child of `pre` or `.code-block`. Use it like this:

    <TIL.DnD>
      <TIL.Dont>
        ```css {copy:false} {footer:false}
        :not(pre), :not(.code-block) {
          > code {
            background-color: 'red';
          }
        }
        ```
      </TIL.Dont>
      <TIL.Do>
        ```css {copy:false} {footer:false}
        :not(pre, .code-block) {
          > code {
            background-color: 'red';
          }
        }
        ```
      </TIL.Do>
    </TIL.DnD>

    ## Turborepo dependsOn

    <TIL.ItemTags>
      <TIL.ItemTag>turborepo</TIL.ItemTag>
      <TIL.ItemTag>monorepo</TIL.ItemTag>
    </TIL.ItemTags>

    ```json {copy:false} {footer:false}
    "build": {
      "dependsOn": ["^lint", "test"]
    },
    ```

    `dependsOn` with ^ means a task from another package, while without ^ means a task from the same package.

    Example: when building app A, it must run the `lint` task on all of its dependency packages, and also run its own `test` task.

  </TIL.Item>
