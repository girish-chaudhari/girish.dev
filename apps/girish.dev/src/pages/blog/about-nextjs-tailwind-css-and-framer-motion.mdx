## Next.js

Before this, I used to create React websites using CRA (Create React App). I switched to Next.js for this project.

Honestly, Next.js is incredibly convenient. It offers all the features I need out-of-the-box, has great documentation, and the developer experience is smooth.

Some standout features:

### Routing

Routing is built-in, including dynamic routes which are perfect for blogs and similar use-cases.

My favourite part? **API Routes**. You can create API endpoints directly without needing to set up Express.

### MDX Support

Very easy to integrate Markdown and MDX using `@next/mdx`, perfect for blog content or even standard pages.

### Optimization

Fonts and images are optimised by default. You can import Google Fonts easily using `@next/font`, and the Image component provides built-in compression and responsive handling.

Plus, other features like fast compiling, i18n routing, and static HTML export are super handy.

Explore their [documentation](https://nextjs.org/docs/getting-started) or just try it out.
